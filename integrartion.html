Adding Ludeo to Your Game

🎮 A quickstart guide on integrating your Game with Ludeo, so your game can recreate specific scenarios to be used as a challenge for a more engaged gaming community.

Dependencies

Ludeo Development Kit utilizes the following dependencies:

1. Newtonsoft

Newtonsoft is employed to serialize and deserialize any.NET object within a JSON. LudeoDK currently uses version 11.0.2 of Newtonsoft.Json.

2. Websocket-Sharp

Websocket-Sharp is used to generate all MonoDevelop configurations. LudeoDk currently uses version 1.0.1 of WebSocketSharp-netstandard.

3. Unity3D

Unity3D is an in-built Unity file that stores a 3D game intended to be played in the Unity Web Player. LudeoDK currently uses version 1.7.0 of Unity3D.

4. MonoBehaviour

MonoBehaviour is the base class from which all Unity scripts are derived. It provides life cycle functions that simplify the app and game development.

Setup and Install

Prerequisites

Before integrating your game with Ludeo, please ensure that you have completed the following:

  • Unity 2019 or a later version.
  • Obtain gameId and apiKey from the developers of the Ludeo SDK.
  • Gain access to the Unity Ludeo Development Kit Project here.
  • Get Player Steam Id from your Steam game launcher.

Integration

To integrate your game with Ludeo Development Kit, follow the steps below:

Step 1: Set Steam Player ID

To begin the integration, you must set your Steam Player ID, which you got from the Steam Game Launcher, by following the steps below:

  • Head over to your project.
  • Select Assets > LudeoNonBuild.
  • Select Editor.
  • Open the PlatformSetter with your Code Editor.
  • Head to TestSettingsPerUser method.
  • Add your Steam Player Id by using the code below:
public static TestsSettings <yourPlayerUsername> = new TestsSettings() { steamId = <yourSteamId> };

Step 2: Set API Key

After you've entered your user ID, you'll need to enter your account API Key, which you received from the Ludeo SDK developers, by following the steps below:

  • Head over to your project.
  • Select Assets > LudeoNonBuild.
  • Select Editor.
  • Open the PlatformSetter with your Code Editor.
  • Head to TestSettingsPerGame method.
  • Add your apiKey and specify the game name by using the code below:
public static TestsSettings <yourGameName>= new TestsSettings() { gameName = <yourGameName>,apiKey = <yourApiKey>, ludeoId = "" };

Step 3: Set Game ID

Follow the steps below to set the gameId you obtained from the developers of the Ludeo SDK:

  • Head over to your project.
  • Select Assets > LudeoSDK.
  • Click Resource > LudeoSettings.
  • Set your gameId and Game Name.

Step 4: Build and Run the Project

After configuring every part of the integration, you must build and run the project to ensure that the configurations are correct by following the steps below:

Note
Ensure that you have already chosen Mono in the Scripting Backend component, as LudeoDK employs the MonoBehaviour.
  • Click File tab.
  • Select Build Settings.
  • Click Player Settings > Other Settings.
  • Head to the Configuration section.
  • On Scripting Backend > select Mono.
  • Close the settings > click Build and Run.

How to Install Sample App

After downloading or cloning the Github repo, open the project using the Unity application. Follow the steps below:

Note

  • Extract the **.zip** files.
  • Open the Unity Hub.
  • Locate the project on your device.
  • Open the Project by clicking the Open button.

Step 3: Start the Sample Application

Wait for all of the assets in the Unity hub to be loaded, and follow the steps below:

  • Select Assets.
  • Click the MFPS folder.
  • Click the Scenes folder.
  • Select the MainMenu file.
  • Click the Play button on top of your screen.

Click the Play button on the Sample App main menu to start the game.

User Flows

🚀Learn how the Ludeo's Creator and Player Flows work.

Flow is the procedure or actions that each user takes when using a Ludeo. Creators and players are the two categories of users who interact with Ludeo the most. The Player will re-enact the scenario created by the Creator, while the Creator will create a Ludeo based on the gameplay captured with the LudeoDK. Below are the details of the Creator and Player flows:

Creator Flow

Step 1: Creator Launch the Game

The first step for Creator is to launch the game that has been integrated with Ludeo.

Step 2: Play the Game

Once everything setup, the creator can start the game by following the steps below:

  • Begin a Ludeo session with the creator's settings by callong the BeginGameplay method.
public static LudeoMessage BeginGameplay()
{
if (IsInitDone())
{
ActionBuilders.RequestBeginGameplay(m_systemData);
return m_flowActions.handleBeginSession();
}
return LudeoMessage.Error_Need_Init_First;
}

LudeoDk will return a message if there is an error with the session.

  • Creator set the gameplay state by using the method below:
public static LudeoMessage SetGameplayDefinitions(GameplayDefinitions gameplayDefinitions)
{

if (Tracker.IsReadyToSetDefinitions())

Step 3: Capture the Highlight

Once the creator is in the game, they can capture the gameplay. The saved media will be marked as a highlight by calling the method below:

public static LudeoMessage MarkHighlight()
{
LudeoFlowState state = m_systemData.ludeoState.flowState;
if (IsInitDone())
{
if (state == LudeoFlowState.GameplayOn)
{
HighlightManager.Instance.HighlightLudeo();
ActionBuilders.RequestMarkHighlight(m_systemData);
return LudeoMessage.Success;
}
else
return LudeoMessage.Error_Tracking_Only_During_Gameplay;
//return Tracker.MarkHighlight();
}
return LudeoMessage.Error_Need_Init_First;
}
