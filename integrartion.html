Adding Ludeo to Your Game

üéÆ A quickstart guide on integrating your Game with Ludeo, so your game can recreate specific scenarios to be used as a challenge for a more engaged gaming community.

Setup and Install

Prerequisites

Before integrating your game with Ludeo, please ensure that you have completed the following:

  • Unity 2019 or a later version.
  • Obtain gameId and apiKey from the developers of the Ludeo SDK.
  • Get the LudeoDK Unity Package.
  • Get Player Steam Id from your Steam game launcher.

Integration

To integrate your game with our Ludeo Development Kit, follow the steps below:

Step 1: Import LudeoDK Unity Package

Import the LudeoDK Unity package to your project, by following the steps below:

  • Click Assets.
  • Click Import Package > Custom Package.
  • Select the LudeoDK Unity package on your device.
  • Click Import.

Step 2: Integrate Your Game with Our SDK

After importing our LudeoDK Unity package, you must follow the steps below to integrate your game with our SDK:

  • Select LudeoSDK > Resources.
  • Click LudeoSettings.
  • Enter your Game Id and Game Name.
  • Save your project.

Step 3: Build and Run Your Project

After integrating your game with our SDK, you have to build and run your project to ensure a successful integration by following the steps below:

  • Click File > Build and Run.

User Flows

üöÄLearn how the Ludeo's Creator and Player Flows work.

Flow is the procedure or actions that each user takes when using a Ludeo. Creators and players are the two categories of users who interact with Ludeo the most. The Player will re-enact the scenario created by the Creator, while the Creator will create a Ludeo based on the gameplay captured with the LudeoDK. Below are the details of the Creator and Player flows:

Creator Flow

private void Use_Example_SetGameplayDefinitions()
{
var gameplayDefinitions = new GameplayDefinitions();
gameplayDefinitions.AddDefinition("PlayerClass", (int)PlayerClass.Assault);
gameplayDefinitions.AddDefinition("SceneName", "ExampleLevel");
gameplayDefinitions.AddDefinition("GameMode", (int)GameMode.TDM);
gameplayDefinitions.AddDefinition("RoundStyle", (int)RoundStyle.Rounds);
gameplayDefinitions.AddDefinition("MapName", "Guanaco Desert");
gameplayDefinitions.AddDefinition("Goal", 1000);
gameplayDefinitions.AddDefinition("FriendlyFire", true);
gameplayDefinitions.AddDefinition("MaxPlayers", 5);
int i = 0;
foreach (var playerData in LudeoGameManager.RemotePlayersData)
{
var pos = playerData.V3Position;
gameplayDefinitions.AddDefinition("RemotePlayerName_" + i, "BOT_" + (i + 1));
gameplayDefinitions.AddDefinition("RemotePlayerTeam_" + i, 2);
i++;
}
gameplayDefinitions.AddDefinition("LocalPlayerTeam", 1);
gameplayDefinitions.AddDefinition("TotalRemotePlayers", 6);
var ok = LudeoManager.SetGameplayDefinitions(gameplayDefinitions);
if (ok == LudeoMessage.Success)
ok = LudeoManager.ReadyForGameplay();
Debug.Log("[Response] - SetGameplayDefinitions = " + ok);
}

This will return a message if there is an error and a Tracker object if successful.

private void Use_Example_GetGameplayDefinitionsKeys()
{
string[] keys;
LudeoManager.GetGameplayDefinitionsKeys(LudeoParam.Int, out keys);
foreach (var k in keys)
{
LudeoManager.GetGameplayDefinition(k, out int v);
Debug.Log("Int key found: " + k + " value is: " + v);
}
LudeoManager.GetGameplayDefinitionsKeys(LudeoParam.Vec3, out keys);
foreach (var k in keys)
{
LudeoManager.GetGameplayDefinition(k, out Vec3 v);
Debug.Log("Vec3 key found: " + k + " value is: " + v.ToString());
}
LudeoManager.GetGameplayDefinitionsKeys(LudeoParam.String, out keys);
foreach (var k in keys)
{
LudeoManager.GetGameplayDefinition(k, out string v);
Debug.Log("String key found: " + k + " value is: " + v);
}
}
