Adding Ludeo to Your Game

üéÆ A quickstart guide on integrating your Game with Ludeo, so your game can recreate specific scenarios to be used as a challenge for a more engaged gaming community.

Dependencies

Currently, Ludeo has three crucial dependencies on the Ludeo Development Kit:

Newtonsoft

Newtonsoft is employed to serialize and deserialize any.NET object contained within a Json. LudeoDK currently uses version 11.0.2 of Newtonsoft.Json.

Websocket-Sharp

Websocket-Sharp is used to generate all MonoDevelop configurations. LudeoDk currently uses version 1.0.1 of WebSocketSharp-netstandard.

Unity3D

Unity3D is an in-built Unity file used to store a 3D game that is intended to be played in the Unity Web Player. LudeoDK currently uses version 1.7.0 of Unity3D.

Setup and Install

Prerequisites

Before integrating your game with Ludeo, please ensure that you have completed the following:

  • Installed Unity 2019 or a later version.
  • Obtain gameId from the developers of the Ludeo SDK.
  • Gain access to the Unity LudeoDK Module here.

Integration

To integrate your game with LudeoDK, follow the steps below:

Step 1: Set Game ID

First, to integrate you need to set your gameId that you have obtained from Ludeo SDK developers on LDKSettings.json or LudeoSettings.json file, by following the steps below:

  • Head over to your project.
  • Select Assets > StreamingAssets.
  • Click LudeoDK > LDKSettings.json.
  • Add the following code:
"gameId": <replace with your game ID you got from Ludeo>

Step 2: Initialize LudeoDK

Before initializing please set a player Steam ID on your LudeoDK config file. Initialize the LudeoDK by calling the method below:

LDKManager.Init(<player steam id>);

User Flows

üöÄLearn how the Ludeo's Creator and Player Flows work.

Player Flow and Creator Flow are the two categories of User Flows that can be found in Ludeo. Player is the one who will re-enact the creator's scenario, while Creator is the one who will create a Ludeo based on the gameplay that had been captured using the LudeoDK. The details of the Creator and Player flows are outlined below:

Creator

Step 1: Creator Launch the Game

The first step for Creator is to launch the game that has been integrated with Ludeo.

Step 2: Play the Game

Once everything setup, the creator can start the game by following the steps below:

  • Begin a Ludeo session with the creator's settings by callong the BeginGameplay method.
public static LudeoMessage BeginGameplay()
{
if (IsInitDone())
{
ActionBuilders.RequestBeginGameplay(m_systemData);
return m_flowActions.handleBeginSession();
}
return LudeoMessage.Error_Need_Init_First;
}

LudeoDk will return a message if there is an error with the session.

  • Creator set the gameplay state by using the method below:
public static LudeoMessage SetGameplayDefinitions(GameplayDefinitions gameplayDefinitions)

Step 3: Capture the Highlight

Once the creator is in the game, they can capture the gameplay. The saved media will be marked as a highlight by calling the method below:

public static LudeoMessage MarkHighlight()
{
LudeoFlowState state = m_systemData.ludeoState.flowState;
if (IsInitDone())
{
if (state == LudeoFlowState.GameplayOn)
{
HighlightManager.Instance.HighlightLudeo();
ActionBuilders.RequestMarkHighlight(m_systemData);
return LudeoMessage.Success;
}
else
return LudeoMessage.Error_Tracking_Only_During_Gameplay;
//return Tracker.MarkHighlight();
}
return LudeoMessage.Error_Need_Init_First;
}
