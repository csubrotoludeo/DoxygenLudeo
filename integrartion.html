Setup and Install

Prerequisites

Before integrating your game with Ludeo, please ensure that you have completed the following:

  • Unity 2019 or a later version.
  • Obtain apiKey from the developers of the Ludeo SDK.
  • Get the Ludeo SDK Unity Package.
  • Get Game Steam Id from your Steam game launcher.

Add The Ludeo Unity Package To Your Project

1. Download the LudeoSDK Unity package.
2. Import the Unity package into your project.

User Flows

üöÄLearn how the Ludeo's Creator and Player Flows work.

Flow is the procedure or actions that each user takes when using a Ludeo. Creators and players are the two categories of users who interact with Ludeo the most. The Player will re-enact the scenario created by the Creator, while the Creator will create a Ludeo based on the gameplay captured with the Ludeo SDK. Below are the details of the Creator and Player flows:

Creator Flow

Creator Flow is the flow which enables Ludeos creation. Integrate your game with it by following these steps:

1. On receiving WaitingForLoadGameplayData and when your are ready call:

LudeoManager.GameplayDataReady()

    See GameplayDataReady() for more info.

2. On receiving WaitingForSetGameplayDefinitions call:

LudeoManager.SetGameplayDefinitions(GameplayDefinitions)

    See SetGameplayDefinitions(GameplayDefinitions) for more info.

3. On receiving WaitingForReadyForGameplay and when your gameplay is ready to start call:

LudeoManager.ReadyForGameplay()

    For instance: when you are done loading the game data and before showing your game match countdown.

    See ReadyForGameplay() for more info.

4. On receiving WaitingForGameplayBegin and when your gameplay is beginning or counting down before beginning as in the above example call:

LudeoManager.BeginGameplay()

    See BeginGameplay() for more info.

5. Now that your game is running you can feed the LudeoSDK with any game state variables which you want in order to create Ludeos using:

LudeoManager.SetGameplayVariable(string, value)

    See SetGameplayVariable(string, value) for more info.

6. To capture a Ludeo highlight at any given time during your gameplay call:

LudeoManager.MarkHighlight();

    See MarkHighlight() for more info.

7. When your gameplay run is over (player dies, round is over etc), you need to notify the LudeoSDK by calling:

LudeoManager.EndGameplay();

    See EndGameplay() for more info.

8. When the LudeoSDK has done processing this creation session it will fire WaitingForLoadGameplayData again (step 1) and you can run another session or quit.

Note
Calling MarkHighlight() sets the last time frame as a highlight until the moment of calling it.
Note
Calling AbortGameplay() will forefit any existing LudeoSDK flow.
Note
All unique string keys in Ludeo are limited to 32 characters.
Note
At EndGameplay(), LudeoSDK uploads any marked highlights to the cloud.

Player Flow

Player Flow is the flow which enables Ludeos creation. Integrate your game with it by following these steps:

1. To start your game to replay a Ludeo, use the steam browser protocol, specifically the rungameid command, followed by the Ludeo id as a parameter. The end result looks like: steam://rungameid/<steam_app_id>/ludeoid%20<ludeo id> ( %20 is a space in the url). Please provide Ludeo with your steam app id.

2. In order to obtain the Ludeo id to be played, you will need to process the incoming parameter in your game: Go to your steam dashboard -> your application -> edit steamworks settings -> Installation tab -> General Installation -> Tick the checkbox of Enable Using ISteamApps::GetLaunchCommandLine(). In your game use ISteamApps::GetLaunchCommandLine(), splitting the argument by the space. For example:

const char SPLIT_CHAR = ' ';
const int COMMAND_LINE_STRING_BUFFER_SIZE = 260;
string commandLineArgs;
SteamApps.GetLaunchCommandLine(out commandLineArgs,
COMMAND_LINE_STRING_BUFFER_SIZE);
string[] args = commandLineArgs.Split(SPLIT_CHAR);

3. Now that you‚Äôve followed the previous steps, when receiving the WaitingForLoadGameplayData, you have the Ludeo Id so you can use it to call:

LudeoManager.GameplayDataReady(Guid ludeoId)

    See GameplayDataReady(Guid) for more info

4. On receiving WaitingForGetGameplayDefinitions call:

LudeoManager.GetGameplayDefinitionsKeys(LudeoParam paramType, out
string[] keys)

and:

LudeoManager.GetGameplayDefinition(LudeoParam paramType, out value)

    These will give you the definitions kept in a previously created and downloaded Ludeo.

    See GetGameplayDefinitions(LudeoParam, out string[]) for more info.

    See GetGameplayDefinition(LudeoParam, out value) for more info.

5. On receiving WaitingForReadyForGameplay and when your gameplay is ready to start call:

LudeoManager.ReadyForGameplay()

    For instance: when you are done loading the game data and before showing your game match countdown.

    See ReadyForGameplay() for more info.

6. When playing a Ludeo, LudeoSDK will popup a Ludeo objective screen. Once the player has clicked the button to Start Ludeo, LudeoSDK will fire the state in the next step listed below.

7. On receiving WaitingForGameplayBegin and when your gameplay is beginning or counting down before beginning as in the above example call:

    See BeginGameplay() for more info.

8. Now that your game is running you should feed the LudeoSDK with the same game state variables you have used in the Creator Flow so that Ludeo tracks the Ludeo game progress:

    See SetGameplayVariable(string, value) for more info
