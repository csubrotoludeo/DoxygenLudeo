How It Works?

Ludeo creates LudeoDK, which stands for the Ludeo Development Kit. This facilitates the integration between the Ludeo system and the chosen game, allowing Ludeo to recreate the highlighted moment within the game. LudeoDK was created through the use of a Software Development Kit.

Why use SDK?

SDK enables Ludeo to construct the LudeoDK application, which functions as a toolkit and enables developers to integrate with the chosen game. LudeoDK is exclusive to Ludeo and enables users to recollect, recreate, and experience the exact highlight moment of other players.

Benefits of SDK

Ludeo utilizes SDK based on the following factors:

  • Easy Integration: By utilizing the LudeoDK, the integration process is sped up because it simplifies integrating with the Ludeo technology infrastructure and requires minimal maintenance.
  • Performance: LudeoDK enhances the experience without affecting your game's FPS, allowing you to relive the gaming moment without FPS drops or latency.
  • Privacy: Using GDPA Compliance and COPPA Compliance, LudeoDK also helps you maintain your privacy.
  • Security: Using MD5 Encryption, RSA, PEM Keys, HTTPS, and SSL/TLS to maintain security, LudeoDK is extremely secure.

How Ludeo Works

First Step

Based on the Ludeo Architecture after a game already integrated with LudeoDK, a player will play a game and then activate the LudeoDK to record gameplay. LudeoDK will use the Game Tracker and Media Uploader components to record the gameplay and the gameplay's essential statistics, which will then be sent to the Ludeo Backend Services.

Second Step

On Ludeo Backend Services, the Data Collector calculates and filters key info from the Game Tracker that can be used for Ludeo's goal and scoring parameters. The Media Server will temporarily store the media files uploaded by the Media Uploader, such as videos and images. The Game Tracker and Media Server then transmit the collected data to the Game Manger, which stores all filtered data.

Third Step

Creator Lab receives essential gameplay information from the Game Manger, which it then uses to determine the goal and score parameters for the creator to select. This is the stage in which the creator configures their Ludeo and chooses the Ludeo's Objective and Score Parameters on the Creator Lab.

Fourth Step

The gameplay that has been configured and converted into a Ludeo will be saved to the Ludeo Manger. The Ludeo can then be converted into a link that can be shared on any social media platform or uploaded directly to the Ludeo Discovery Platform. The Ludeo Manger then sends the Ludeo to the Ludeo Player on the Ludeo Backend Services.

Fifth Step

The Ludeo Player on the Ludeo Backend Services then transmits the configured Ludeo data to the Ludeo Player on the game that has been integrated with the LudeoDK; this makes the configured Ludeo data playable. Therefore, if other players click the links or play button on the Ludeo Discovery Platform, it will transport them to the steam launcher and launch the respective game and create a game mode similar to the gameplay, along with the goals and score parameters that the creator has set.
